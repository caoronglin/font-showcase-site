## 核心技术特性

### 1. Simplex 噪声流场

- **改进的 Simplex 噪声算法**：原生实现，零依赖
- **分形布朗运动（FBM）**：4 层八度叠加，创造丰富的流动纹理
- **参数可调**：持久度 0.5、间隙度 2.0、时间演化速度 0.15
- **固定种子**：使用种子 12345，保证动画可重现

### 2. 粒子系统

- **数量自适应**：根据画布宽度动态调整（50-150 个）
- **墨量衰减**：每帧衰减 0.002，模拟墨水干燥过程
- **边缘重生**：粒子超出边界或墨量耗尽时从边缘重新生成
- **历史轨迹**：每个粒子保存最近 5 帧位置，绘制平滑拖尾

### 3. 鼠标交互

- **双模式交互**：
  - **慢速移动**：吸引模式（模拟"按笔"）
  - **快速移动**：排斥模式（模拟"提笔"）
- **速度阈值**：2 像素/帧区分快慢
- **高斯衰减**：作用力随距离平滑衰减
- **触摸支持**：移动设备可通过触摸互动

### 4. 性能优化

- **离屏 Canvas**：
  - 粒子轨迹和连接线绘制到离屏 buffer
  - 每 3 帧同步一次到主画布
  - 减少重复绘制开销

- **空间哈希**：
  - 单元格大小 40px
  - 将 O(n²) 距离检测降至 O(n log n)
  - 只检测相邻单元格内的粒子

- **节流更新**：
  - 流场每 3 帧更新一次（而非每帧）
  - 页面不可见时自动暂停动画
  - 支持 `prefers-reduced-motion`

### 5. 入场动画

- **渐进式显现**：粒子在 2.5 秒内分批出现
- **交错延迟**：每个粒子间隔 30ms 激活
- **三次缓动**：使用 easeOutCubic 缓动曲线
- **墨量渐变**：从 0 渐显到正常不透明度

### 6. 暗色模式适配

- **自动检测**：监听 `prefers-color-scheme` 和 `.dark` 类
- **颜色缓存**：避免每帧查询 CSS 变量
- **双模式配色**：
  - 浅色：粒子 `rgba(80,80,80)`, 连线 `rgba(0,0,0)`
  - 暗色：粒子 `rgba(180,180,180)`, 连线 `rgba(255,255,255)`

## 设计系统整合

### 使用的设计令牌

代码完全遵循项目的 `variables.css` 设计系统：

- **颜色**：响应 `--ink-*` 系列变量
- **间距**：配置参数使用逻辑像素值
- **过渡**：入场动画使用 2500ms（接近 `--transition-slower`）
- **阴影**：虽然 Canvas 不使用阴影，但整体视觉层次与 `--shadow-*` 系统协调

### 墨韵美学

整体视觉风格呼应设计系统中的"墨韵 Ink Harmony"主题：

- 粒子如水墨滴落宣纸，自然晕染
- 流动感模拟书法运笔的气韵
- 克制的配色（黑白灰）避免喧宾夺主
- 留白与密度的对比营造空间感

## 配置参数说明

### 粒子系统配置

```javascript

  minCount: 50,           // 最小粒子数
  maxCount: 150,          // 最大粒子数
  baseSize: 2,            // 基础大小 (px)
  sizeVariance: 1.5,      // 大小变化范围
  minSpeed: 0.2,          // 最小速度
  maxSpeed: 1.5,          // 最大速度
  inkDecay: 0.002,        // 墨量衰减率
  respawnMargin: 50       // 重生边距 (px)

  octaves: 4,             // 噪声层数
  persistence: 0.5,       // 持久度

  timeScale: 0.15,        // 时间演化速度
  zOffset: 100            // 噪声 Z 轴偏移
}
```

## 性能基准

### 目标性能

- **帧率**：60 FPS（高端设备）
- **帧率**：30+ FPS（中端设备）
- **内存**：< 50MB
- **CPU**：< 20% 单核占用

### 优化措施

1. **粒子数量限制**：最多 150 个，避免过度绘制
2. **空间哈希**：减少不必要的距离计算
3. **离屏渲染**：减少重绘区域
4. **流场节流**：每 3 帧更新一次噪声场
5. **页面可见性**：隐藏时自动暂停
6. **Typed Arrays**：使用 `Uint8Array`、`Float32Array` 存储数据

### 完全支持

- Chrome 80+
- Firefox 75+

- Edge 80+

### 降级策略

- **不支持 Canvas**：静默失败，不影响页面其他功能
- **不支持 requestAnimationFrame**：降级到 `setInterval`
- **prefers-reduced-motion**：减少粒子数量，降低流动速度
- **触摸设备**：支持触摸交互，但禁用复杂的鼠标跟随

## 使用说明

### 基本使用

动画会自动初始化，无需手动调用。只需确保：

```html
<canvas id="hero-canvas" class="hero__canvas"></canvas>
```

### 手动控制

如需手动控制，可导入模块：

```javascript
import { initHeroCanvas } from './scripts/hero-canvas.js';

// 重新初始化（例如窗口大小变化后）
initHeroCanvas();
```

### 参数调优

修改 `CONFIG` 对象中的参数可调整视觉效果：

```javascript
// 在 hero-canvas.js 中修改
const CONFIG = {
  particle: {
    maxCount: 200,  // 增加粒子密度
    inkDecay: 0.001 // 减慢墨量衰减
  }
};
```

## 设计理念

> **算法即艺术，代码即笔墨**

墨韵流形证明了计算过程本身可以承载文化内涵。通过 Simplex 噪声和粒子系统的组合，我们创造出了一个既有数学精确性又有艺术表现力的动态背景。

这种设计方法呼应了中国传统美学的核心理念：

- **气韵生动**：流场驱动粒子产生有机的流动感
- **虚实相生**：粒子的生死循环创造虚实变化
- **意在笔先**：固定的种子确保每次运行都呈现相同的美感
- **留白之美**：克制的粒子密度营造呼吸空间

---

## 技术亮点

1. **零依赖**：所有算法原生实现，包括 Simplex 噪声
2. **模块化**：类和方法职责清晰，便于维护和扩展
3. **类型安全**：使用 Typed Arrays 保证数据结构一致性
4. **响应式**：自动适应不同屏幕尺寸和 DPI
5. **无障碍**：支持 prefers-reduced-motion，不影响页面可访问性
